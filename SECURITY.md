# Security Policy

## Reporting a Vulnerability

The SkiaSharp Chart Engine team takes security vulnerabilities seriously. If you discover a security vulnerability, **please do not open a public issue**. Instead, use one of the following methods to report it confidentially.

### Reporting Methods

1. **GitHub Private Vulnerability Reporting (Recommended)**
   - Visit: https://github.com/vladyslav-zaiets/skiasharp-chart-engine/security/advisories/new
   - This allows you to report directly within GitHub's secure system

2. **Email**
   - Send details to: rutova2@gmail.com
   - Include "SECURITY" in the subject line

### What to Include

Please provide:
- Description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fix (if you have one)
- Your contact information (optional)

## Response Timeline

We are committed to addressing security vulnerabilities promptly:

- **48 hours**: Initial acknowledgment of your report
- **1 week**: Initial security assessment and plan
- **30 days**: Target for patch release (may vary based on complexity)

## Security Best Practices for Users

When using SkiaSharp Chart Engine in production:

1. Keep your .NET runtime and dependencies up to date
2. Use the latest version of SkiaSharp Chart Engine
3. Validate and sanitize all user inputs before passing to the chart engine
4. Run applications with minimal required permissions
5. Monitor for security advisories

## Contact

For security matters, contact the maintainers at the email address above. For other inquiries, please use standard GitHub Issues.

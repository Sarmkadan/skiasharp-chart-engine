# AlertingService

The `AlertingService` class provides a centralized mechanism for managing monitoring rules and tracking alerts generated within the `skiasharp-chart-engine`. It allows for the definition of condition-based rules, evaluation of these rules, maintenance of alert history, and lifecycle management of alerts through acknowledgement and clearing.

## API

### AlertingService
Initializes a new instance of the `AlertingService`.

### RegisterRule
Registers a new rule for evaluation.
*   **Parameters:**
    *   `name`: The unique identifier for the rule.
    *   `condition`: A `Func<bool>` that returns `true` when the rule is triggered.
    *   `severity`: The `AlertSeverity` level associated with the rule.
    *   `message`: The message to be included when an alert is generated.

### CheckAsync
Evaluates all registered rules asynchronously.
*   **Returns:** A `Task` representing the asynchronous evaluation process.

### AcknowledgeAlert
Marks a specific alert as acknowledged.
*   **Parameters:**
    *   `id`: The unique identifier of the alert.
*   **Returns:** `true` if the alert was found and successfully acknowledged; otherwise, `false`.

### GetActiveAlerts
Retrieves a list of all alerts that are currently active and not yet acknowledged.
*   **Returns:** A `List<Alert>`.

### GetAlertsBySeverity
Retrieves active alerts filtered by a specific severity level.
*   **Parameters:**
    *   `severity`: The `AlertSeverity` level to filter by.
*   **Returns:** A `List<Alert>`.

### ClearAcknowledgedAlerts
Removes all acknowledged alerts from the active alert collection.
*   **Returns:** The number of alerts cleared.

### GetAlertHistory
Retrieves the complete history of all alerts generated by the service.
*   **Returns:** A `List<Alert>`.

### GetStatistics
Retrieves summary statistics for the service.
*   **Returns:** An `AlertStatistics` object.

## Related Types

### Alert
Represents a triggered issue within the system.
*   **Id**: `string` - The unique identifier.
*   **Message**: `string` - The descriptive message.
*   **Severity**: `AlertSeverity` - The severity level.
*   **Timestamp**: `DateTime` - When the alert was generated.
*   **Acknowledged**: `bool` - The current acknowledgement status.
*   **AcknowledgedAt**: `DateTime?` - The time of acknowledgement, if applicable.

### Rule
Represents the definition for a monitored condition.
*   **Name**: `string` - The identifier for the rule.
*   **Condition**: `Func<bool>` - The predicate function for evaluation.
*   **Message**: `string` - The alert message upon trigger.
*   **Severity**: `AlertSeverity` - The severity of the generated alert.

## Usage

### Registering and Evaluating a Rule
```csharp
var alertingService = new AlertingService();

// Register a rule that triggers when a value exceeds a threshold
alertingService.RegisterRule("ThresholdExceeded", () => currentValue > 100, AlertSeverity.Warning, "Value threshold exceeded.");

// Evaluate rules
await alertingService.CheckAsync();
```

### Managing Alerts
```csharp
// Retrieve active alerts
var activeAlerts = alertingService.GetActiveAlerts();

foreach (var alert in activeAlerts)
{
    // Acknowledge the first active alert
    if (alertingService.AcknowledgeAlert(alert.Id))
    {
        Console.WriteLine($"Alert {alert.Id} acknowledged.");
    }
}

// Clean up acknowledged alerts
int clearedCount = alertingService.ClearAcknowledgedAlerts();
```

## Notes

*   **Thread-Safety**: The `AlertingService` is not inherently thread-safe. Concurrent access to rule registration, evaluation, or alert management methods from multiple threads should be synchronized externally.
*   **Evaluation Frequency**: The `CheckAsync` method performs evaluation based on the conditions defined at the time of rule registration. It is intended to be called at an appropriate frequency based on the application's monitoring requirements.
*   **Alert Lifecycle**: Alerts transition from active to acknowledged via `AcknowledgeAlert`. Once acknowledged, they remain in the history until explicitly removed, though `ClearAcknowledgedAlerts` facilitates removal from the active set.

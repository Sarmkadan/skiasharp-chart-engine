# TransitionEasingCalculator

The `TransitionEasingCalculator` provides functionality for interpolating chart data between defined states over time. It processes `TransitionKeyframe` definitions to compute a series of `TransitionRenderFrame` objects, which are encapsulated within a `TransitionResult` to facilitate smooth, performant visual transitions in chart rendering engines.

## API

### TransitionEasingCalculator

*   `public static double Calculate`: Performs the easing calculation to determine an interpolated value at a specific point in the transition timeline.

### TransitionKeyframe

*   `public sealed record TransitionKeyframe`: Represents a discrete point in a transition, defining the desired state at a specific time.

### TransitionRenderFrame

*   `public sealed record TransitionRenderFrame`: Represents an intermediate state generated by the calculator, ready for use by the rendering pipeline at a specific time interval.

### TransitionResult

*   `public required string ChartId`: Identifier for the chart associated with the transition result.
*   `public bool Success`: Indicates whether the transition calculation was completed successfully.
*   `public IReadOnlyList<TransitionRenderFrame> Frames`: The collection of calculated render frames generated during the transition.
*   `public double TotalDurationMs`: The total calculated duration of the transition in milliseconds.
*   `public long RenderTimeMs`: The time taken to execute the transition calculation, in milliseconds.
*   `public string? ErrorMessage`: Contains descriptive error information if the transition calculation failed.
*   `public Exception? Exception`: The exception object generated during the calculation, if a failure occurred.
*   `public IReadOnlyDictionary<string, object> Metadata`: A collection of additional metadata associated with the transition result.
*   `public static TransitionResult CreateSuccess`: Factory method used to instantiate a `TransitionResult` representing a successful calculation.
*   `public static TransitionResult CreateFailure`: Factory method used to instantiate a `TransitionResult` representing a failed calculation.
*   `public override string ToString`: Returns a string representation of the `TransitionResult` object.

## Usage

### Example 1: Successful Transition Calculation

```csharp
// Defining keyframes for a chart transition
var startFrame = new TransitionKeyframe();
var endFrame = new TransitionKeyframe();

// Calculating the transition result
var result = TransitionEasingCalculator.Calculate(new[] { startFrame, endFrame });

if (result.Success)
{
    Console.WriteLine($"Transition completed in {result.RenderTimeMs}ms. Generated {result.Frames.Count} frames.");
}
```

### Example 2: Manual TransitionResult Creation

```csharp
// Manually creating a failed transition result for error reporting
var result = TransitionEasingCalculator.CreateFailure(
    "chart-001",
    "Invalid keyframe sequence",
    new ArgumentException("Keyframe duration cannot be negative.")
);

if (!result.Success)
{
    Console.Error.WriteLine($"Error in chart {result.ChartId}: {result.ErrorMessage}");
}
```

## Notes

*   **Thread Safety**: The `TransitionEasingCalculator.Calculate` method is stateless and thread-safe, provided the input keyframe collections are not modified during execution. `TransitionResult`, `TransitionKeyframe`, and `TransitionRenderFrame` are immutable records, ensuring thread-safe access to their properties.
*   **Edge Cases**: `Calculate` may throw exceptions if provided with null input collections or if the keyframe data contains invalid durations that prevent interpolation. Ensure input validation is performed before invoking the calculation.
*   **Performance**: `RenderTimeMs` is provided in the `TransitionResult` to allow for monitoring the performance impact of complex transition interpolations. High-frequency updates or large numbers of keyframes may increase the `RenderTimeMs` and impact UI responsiveness.
